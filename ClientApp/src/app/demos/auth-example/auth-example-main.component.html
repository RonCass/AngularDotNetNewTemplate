Auth Example

This demo requires that you be running it locally with the database created that contains the users.
This demo uses:
  • home/login/login.component.ts: The LoginComponent will call back to the AuthController and get JWT auth token that will be used on future calls to the backend API that should be hidden behind an [Authorize] decorator on the contollers.
  • core/services/current-user.service.ts: The CurrentUserService will get and set the user and token information to LocalStorage in the browser. Both objects are BehaviorSubjects (Subjects from RxJs) ($tokenInfo and $currentUserInfo) that when changed will allow for other areas of the site that need this information to subscribe to it.
  • core/services/auth-guard.service.ts: The AuthGuardService protects routes so that only Authorized Users can use routes. This service checks CurrentUserService to see if the user is authenticated when they click a link for a route that is protected with the AuthGuardService by putting a "canActivate: [AuthGuard]" on the route itself.
